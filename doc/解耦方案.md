# 解耦方案：将用户可编辑部分从底层引擎中分离

## 1. 问题分析

当前代码中，用户可编辑内容（HTML/CSS）和交互逻辑与底层引擎高度耦合：

| 耦合点 | 位置 | 描述 |
|:--|:--|:--|
| HTML 内容 | `demo_main.c` | 整个页面 HTML 硬编码为字符串常量 |
| CSS 样式 | `demo_main.c` | 样式表硬编码为字符串常量 |
| 按钮行为 | `render.h` | `handle_button_click()` 硬编码匹配按钮文本（"提交"/"重置"/"清空A"等） |
| 输入缓冲 | `render.h` | `g_input_buf[256][64]` 静态全局数组 |
| 状态消息 | `render.h` | `g_status_msg[256]` 静态全局变量 |
| 交互状态 | `styletree.h` + `render.h` | `g_interact_hover/focus/active` 跨模块全局变量 |
| 所有逻辑 | `demo_main.c` | 解析→样式→布局→渲染→交互 全在 ~140 行入口中 |

用户想改页面内容或交互行为，就必须修改 C 源码然后重新编译，无法独立配置。

## 2. 设计目标

1. **用户可编辑部分独立成文件** — 用户只需关注一个文件，不改引擎
2. **交互回调可配置** — 按钮行为通过回调函数注册，而非硬编码匹配
3. **引擎层不变** — `gumbo.h` / `katana.h` / `termbox2.h` / `uc.h` 不动
4. **保持 tcc 单文件编译** — 仍用 `tcc src/demo_main.c -I src -run`
5. **不破坏现有功能** — 所有交互测试、表格、样式、渲染保持不变

## 3. 架构设计

```
┌─────────────────────────────────────────────┐
│  用户页面层 (demo_page.h)                      │  ← 用户可编辑
│  - HTML 内容                                  │
│  - CSS 样式                                   │
│  - 按钮/交互回调实现                           │
│  - 页面状态管理                                │
├─────────────────────────────────────────────┤
│  交互框架层 (core/interact.h) [NEW]          │  ← 从 render.h 提取
│  - InteractCallbacks 回调接口定义              │
│  - 焦点遍历 / 输入编辑 / 事件分发              │
│  - 通用交互循环 (无需用户修改)                 │
├─────────────────────────────────────────────┤
│  渲染引擎层 (core/render.h)                   │  ← 精简版
│  - Screen 缓冲管理                            │
│  - 节点渲染 / 边框 / 表格网格                  │
│  - render_init/shutdown/size                  │
├─────────────────────────────────────────────┤
│  布局/样式引擎 (core/layout.h / styletree.h)   │  ← 不变
│  - 样式树构建 / 选择器匹配                    │
│  - 布局计算 / Box Model / Flex               │
├─────────────────────────────────────────────┤
│  基础库 (core/gumbo.h/katana.h/termbox2.h/uc.h)│
└─────────────────────────────────────────────┘
```

## 4. 接口设计

### 4.1 `InteractCallbacks` 回调接口 (interact.h)

```c
typedef struct {
    // 按钮点击回调: btn_text 是按钮的文本内容
    // focus_list/focus_count 是当前所有可聚焦元素
    // input_buf 是输入框缓冲区数组
    // 返回 true 表示需要重建布局树
    bool (*on_button_click)(const char* btn_text,
                            int focus_idx,
                            LayoutNode** focus_list, int focus_count,
                            char input_buf[][64], int input_buf_count,
                            void* user_data);
    
    // 状态消息更新回调 (可选, 为 NULL 则使用默认状态栏)
    void (*set_status)(const char* msg, void* user_data);
    
    // 用户自定义数据指针
    void* user_data;
} InteractCallbacks;
```

### 4.2 `render_run_ex` 新签名 (interact.h → 从 render.h 移入)

```c
// 运行交互循环; 接受回调参数替代硬编码行为
void interact_run(LayoutNode* root, KatanaStylesheet* css,
                  StyledNode* styled_root,
                  InteractCallbacks* callbacks);
```

### 4.3 用户页面定义 (demo_page.h)

```c
// 用户在此定义页面内容
#define DEMO_HTML  "<html>...</html>"
#define DEMO_CSS   "body{...}"

// 用户在此实现按钮回调
static bool demo_on_button_click(const char* btn_text, ...) {
    // 自定义按钮行为
}

// 用户页面初始化：设置回调并启动
void demo_run(void);
```

## 5. 实现步骤

### Step 1: 创建 `core/interact.h`

从 `render.h` 提取以下内容：
- `InteractCallbacks` 结构体定义
- `render_run_ex()` → 重命名为 `interact_run()`，增加 `InteractCallbacks*` 参数
- `interact_run` 中调用 `callbacks->on_button_click` 替代硬编码的 `handle_button_click`
- 移除 `g_input_buf` / `g_status_msg` / `handle_button_click`
- 所有状态通过 `InteractCallbacks.user_data` 传递

### Step 2: 创建 `src/demo_page.h`

包含：
- `DEMO_HTML` 和 `DEMO_CSS` 宏定义（从 `demo_main.c` 搬过来）
- `demo_on_button_click()` 回调实现（从 `render.h` 的 `handle_button_click` 搬过来）
- 页面状态变量（`g_input_buf`, `g_status_msg` 等）
- `demo_run()` 函数：解析 HTML/CSS → 构建样式/布局 → 调用 `interact_run()`

### Step 3: 精简 `src/core/render.h`

- 删除 `handle_button_click`
- 删除 `g_input_buf`, `g_input_buf_count`, `g_status_msg`
- 删除 `render_set_status`
- 删除 `render_run_ex`
- 保留：`Screen` / `screen_*` / `render_init/shutdown/size`

### Step 4: 精简 `src/demo_main.c`

- 只保留 `#define XXX_IMPLEMENTATION`
- 包含 `demo_page.h`
- 调用 `demo_run()`

## 6. 关键变更对比

### demo_main.c 变化

**之前**: 140+ 行，包含 HTML/CSS 字符串 + 所有初始化
```c
#define UC_IMPLEMENTATION
#include "core/uc.h"
#define STYLETREE_IMPLEMENTATION
#include "core/styletree.h"
// ... 8 个 include + ~100 行 HTML/CSS 字符串 + ~30 行 main()
```

**之后**: ~30 行，纯组装
```c
#define UC_IMPLEMENTATION
#include "core/uc.h"
#define STYLETREE_IMPLEMENTATION
#include "core/styletree.h"
// ... 所有 IMPLEMENTATION + 包含 demo_page.h
#include "demo_page.h"
int main(void) {
    demo_run();
    return 0;
}
```

### render.h 变化

**之前**: 含 `handle_button_click()` + `g_input_buf` + `g_status_msg` + `render_run_ex`
**之后**: 移除了页面特定逻辑，尺寸减少 ~40%

### 用户编辑流程变化

**之前**: 用户需修改 `demo_main.c` 的 HTML/CSS 字符串，改交互要深入 `render.h`
**之后**: 用户只需编辑 `demo_page.h` 一个文件，改 HTML/CSS 和交互行为
